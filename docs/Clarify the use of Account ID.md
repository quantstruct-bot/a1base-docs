
# Understanding the Account ID

This document clarifies the meaning and usage of the `accountId` parameter used throughout the API.  A clear understanding of the `accountId` is crucial for developers to correctly interact with the API and manage their resources.

## Introduction

The `accountId` is a fundamental identifier within our API. It represents the unique identifier for your organization or business entity within our system.  All API requests that create, retrieve, update, or delete resources are scoped to a specific `accountId`.  Think of it as your organization's namespace within our platform.

## What the Account ID Represents

The `accountId` represents your **organization's unique identifier** within our system. It is *not* a user ID or an individual's identifier.  It's the top-level identifier that groups all your users, resources (like messages, email configurations, etc.), and settings.

## How to Obtain Your Account ID

Your `accountId` is provided to you during the onboarding process.  It is typically included in your welcome email or displayed in the administrative dashboard of our platform. If you cannot locate your `accountId`, please contact our support team.

## Account ID, API Keys, and Secrets

The `accountId` works in conjunction with API keys and secrets to authenticate and authorize your API requests.

*   **`accountId`**: Identifies *which* organization's resources you are accessing.
*   **`x-api-key`**:  Authenticates *who* is making the request.  This key is associated with your account and allows us to verify that the request is coming from a legitimate source.
*   **`x-api-secret`**: Provides an additional layer of security.  The API secret should be treated as a password and kept confidential.

**Important:**  Never share your API key or secret with unauthorized individuals.  Treat them as sensitive credentials.

## Using the Account ID in API Requests

To use the `accountId` in API requests, you need to include it as a path parameter in the URL.  For example:

```
GET /v1/messages/threads/{accountId}/get-all
```

In this example, you would replace `{accountId}` with your actual account ID.  For instance, if your `accountId` is `your_org_123`, the URL would become:

```
GET /v1/messages/threads/your_org_123/get-all
```

**Example Request (using `curl`):**

```bash
curl -X GET \
  'https://api.example.com/v1/messages/threads/your_org_123/get-all' \
  -H 'x-api-key: YOUR_API_KEY' \
  -H 'x-api-secret: YOUR_API_SECRET'
```

**Endpoints Affected:**

Based on the provided OpenAPI specification, the following endpoints use the `accountId` path parameter:

*   `/v1/messages/individual/{accountId}/send`
*   `/v1/messages/group/{accountId}/send`
*   `/v1/messages/send/{accountId}`
*   `/v1/messages/individual/{accountId}/get-details/{messageId}`
*   `/v1/messages/threads/{accountId}/get-recent/{threadId}`
*   `/v1/messages/threads/{accountId}/get-details/{threadId}`
*   `/v1/messages/threads/{accountId}/get-all`
*   `/v1/messages/threads/{accountId}/get-all/{phone_number}`
*   `/v1/emails/{accountId}/create-email`
*   `/v1/emails/{accountId}/send`

## Error Handling

If you provide an incorrect or invalid `accountId`, the API will typically return a `404 Not Found` or `401 Unauthorized` error.  Ensure that you are using the correct `accountId` and that your API key and secret are valid and associated with that account.

## Security Considerations

*   Always protect your `accountId`, API key, and secret.
*   Do not embed these credentials directly in your client-side code.
*   Use environment variables or secure configuration management to store these values.
*   Regularly rotate your API keys and secrets for enhanced security.

By understanding and correctly using the `accountId`, you can effectively manage your resources and interact with our API securely and efficiently.
